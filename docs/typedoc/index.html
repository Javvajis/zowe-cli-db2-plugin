@brightside/db2

Db2 for z/OS Plug-in for CA Brightside CLI

This repository contains a plug-in for CA Brightside that interacts with IBM Db2 for z/OS.

Install the Db2 Plug-in From Packages

Follow these steps:

  1. Issue the following command:
     $ bright plugins install @brightside/db2

  2. Validate the installed plug-in:

     $ bright plugins validate db2

 Validation results for plugin 'db2':
 Successfully validated.

Install the Db2 Plug-in From Source

Download and Build the Source Code

To install the Db2 plug-in to CA Brightside from source, issue the following commands:

  1. Clone this repository to your PC:
     $ **REMOVED**
  2. In a command line, issue
     $ cd brightside-db2
  3. Install dependencies:
     $ npm install
  4. Build the plug-in:
     $ npm run build
  5. Install the plug-in:
     $ bright plugins install /path/to/brightside-db2

  6. Validate the installed plug-in:

     $ bright plugins validate db2

 Validation results for plugin 'db2':
 Successfully validated.

Note: For more information about using the install command, see Manage Plug-ins.

Uninstall the Plug-in From CA Brightside

To uninstall the plug-in from CA Brightside, issue the following command:

$ bright plugins uninstall db2

Note: For more information about using the uninstall command, see Manage Plug-ins.

Setting Up the Db2 Plug-in

Create a Db2 Profile

To create a Db2 plug-in profile gather the following data:

  • The Db2 server host name
  • The Db2 server port number
  • The database name. You can also use the location.
  • The user name
  • The password
  • If your Db2 systems uses a secure connection, you can also provide a SSL/TSL certificate file.

You can get this information by issuing the -DISPLAY DDF Db2 command in the SPUFI. If you are unsure, ask your DBA.

To create a Db2 profile, issue the following command:

$ bright profiles create db2 DB2C -H db1c.example.com -P 4567 -d DB1CPTIB -u ibmuser -p s3cUr3P4ss

Profile created successfully! Path:
/home/user/.brightside/profiles/db2/DB2C.yaml

type:  db2
name:  DB2C
hostname: db1c.example.com
port:  4567
username: securely_stored
password: securely_stored
database: DB2CPTIB


Review the created profile and edit if necessary using the profile update
command.

IBM Db2 for z/OS License File

For connectivity against Db2 for z/OS, the Db2 region should be db2connectactivated or a db2connect license file should be provided. Copy the db2connect license file to the [db2 plugin folder]/node_modules/ibm_db/installer/clidriver/license/ folder.

Examples

The following examples show the Db2 plug-in usage:

  • Default help
  • Call a stored procedure
  • Execute a SQL statement
  • Export a table in SQL format

Default help

$ bright db2 --help

 DESCRIPTION
 -----------

   Interact with Db2 for z/OS

 USAGE
 -----

   bright db2 [group]

 COMMANDS
 --------

   call    (experimental) Call a stored procedure
   execute (experimental) Execute a SQL query
   export  (experimental) Export a table

 GLOBAL OPTIONS
 --------------

   --response-format-json  | --rfj (flag) (optional)

      Produce the command response as a JSON document

   --help  | -h (flag) (optional)

      Display help text

Call a stored procedure

Call a stored procedure that returns a result set.

$ bright db2 call sp "DEMOUSER.EMPBYNO('000120')"
-
EMPNO:  000120
FIRSTNME:  SEAN
MIDINIT:
LASTNAME:  O'CONNELL
WORKDEPT:  A00
PHONENO:  2167
HIREDATE:  1963-12-05
JOB:  CLERK
EDLEVEL:  14
SEX:  M
BIRTHDATE: 1942-10-18
SALARY:  29250.00
BONUS:  600.00
COMM:  2340.00

Call a stored procedure and pass parameters.

$ bright db2 call sp "DEMOUSER.SUM(40, 2, ?)" --parameters 0
- 42

Call a stored procedure and pass a placeholder buffer.

$ bright db2 call sp "DEMOUSER.TIME1(?)" --parameters "....placeholder...."
- 2018-02-31-08.46.15

Execute a SQL statement

Count rows in the EMP table.

$ bright db2 execute sql -q "SELECT COUNT(*) AS TOTAL FROM DSN81210.EMP;"
Result #1
-
TOTAL: 43

Get a department name by ID.

$ bright db2 execute sql -q "SELECT DEPTNAME FROM DSN81210.DEPT WHERE DEPTNO='D01'"
Result #1
-
DEPTNAME: DEVELOPMENT CENTER

Export a table in SQL format

Export projects table and display the generated SQL statements.

$ bright db2 export table DSN81210.PROJ
INSERT INTO DSN81210.PROJ(PROJNO, PROJNAME, DEPTNO, RESPEMP, PRSTAFF, PRSTDATE, PRENDATE, MAJPROJ)
VALUES('AD3100', 'ADMIN SERVICES', 'D01', '000010', 6.50, '1982-01-01', '1983-02-01', NULL)

INSERT INTO DSN81210.PROJ(PROJNO, PROJNAME, DEPTNO, RESPEMP, PRSTAFF, PRSTDATE, PRENDATE, MAJPROJ)
VALUES('AD3110', 'GENERAL AD SYSTEMS', 'D21', '000070', 6.00, '1982-01-01', '1983-02-01', 'AD3100')

INSERT INTO DSN81210.PROJ(PROJNO, PROJNAME, DEPTNO, RESPEMP, PRSTAFF, PRSTDATE, PRENDATE, MAJPROJ)
VALUES('AD3111', 'PAYROLL PROGRAMMING', 'D21', '000230', 2.00, '1982-01-01', '1983-02-01', 'AD3110')

...

Export projects table and save the output to a file.

$ bright db2 export table DSN81210.PROJ --outfile projects-backup.sql

You can also pipe the output to gzip for on-the-fly compression.

$ bright db2 export table DSN81210.PROJ | gzip > projects-backup.sql.gz
